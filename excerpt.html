Under 200k — a field guide to heavy Claude Code sessions

Why this exists.

You already know what the 200k wall feels like. You're forty turns into a refactor, Claude has the full mental model of your codebase, and somewhere around turn forty-one the context bar goes red. Autocompact triggers. Older messages get summarized into oblivion. The design decision you argued out three hours ago? One line that loses the reasoning.

This guide is about not getting there. Not because hitting 200k is a moral failure — it isn't — but because most of the tokens you're burning are noise you never needed to load. The 60k MCP tools you don't use in this session. The skill descriptions that trigger on phrases you never type. The memory files that haven't been relevant in months. The capabilities index that gets re-injected every turn.

This excerpt contains one complete chapter and one script. Together they're enough to diagnose your current setup and put a real context indicator in your terminal.

I.

A context audit you can run in under two minutes.

What the context window actually is

The 200k token window is a shared budget across eight or nine distinct components. You don't control most of them directly. But every turn, Claude Code sends the model a prompt shaped roughly like this:

Component Typical share at 50% full
Messages (active transcript)40–60%
MCP tools (deferred)15–30%
Autocompact buffer (reserved)~15%
Skills3–8%
Memory files2–6%
Custom agents2–5%
System prompt + tools5–8%

The buffer is mandatory overhead — you can't reclaim it. Everything else is fair game.

The audit checklist

Run this on a session that feels heavy. Takes 90 seconds if you know where to look.

  1. 1.
    Check the headline number. The context-window indicator is surfaced via the statusline. If you haven't installed one, skip to Appendix A below and install it now, or use /context if your CLI version supports the slash command. You want one number: percentage of 200k used.
  2. 2.
    Note the top three components by token count. For most sessions at 50%+, Messages and MCP tools dominate. If something else is at the top, you're an outlier and the rest of this chapter is less urgent.
  3. 3.
    If Messages is >60%, that's the transcript leaking. Long conversations accumulate tool results — every file Claude read is still sitting in the transcript verbatim. The fix isn't to be more careful; the transcript is append-only. Start a new session and carry only what matters.
  4. 4.
    If MCP tools is >25%, Chapter II is your chapter. The fix takes five minutes and doesn't require uninstalling anything. It's browser-side, not CLI-side — skip ahead if you're impatient.
  5. 5.
    If Skills + Agents + Memory files combined is >20%, you have declaration bloat. Walk ~/.claude/skills/ and ~/.claude/agents/ — anything you don't recognize, rename with a .disabled suffix and see if you miss it in a week.
  6. 6.
    Note the autocompact-imminent threshold. Claude Code triggers autocompact at ~92% by default. If you're at 85%, you have roughly one heavy turn before you lose context. Wrap up the current subproblem, save key state to a memory file, then start fresh.

What "under two minutes" actually means

Ninety seconds to run /context and read the output. Thirty seconds to mentally flag the top two components. You do not need spreadsheets or screenshots. The audit is useful because it's fast enough to run mid-session when you notice the statusline turning yellow. Any slower and you won't do it.

Across every audit, the same pair dominates: Messages and MCP tool schemas. If your audit shows something else at the top, you're unusual and the remaining chapters are less urgent. If it shows these two, you're normal.

Appendix A

The statusline script.

Drop this at ~/.claude/hooks/statusline.sh, make it executable, and register it in settings.json under the statusLine.command key. Every session will then show a color-tiered context indicator in your terminal — green below 65%, yellow 65–84%, red at 85%+.

The key field — context_window.used_percentage — does not appear in the public Claude Code documentation as of early 2026. It was found by grepping the bundled cli.js directly (Chapter IV of the full guide covers that pattern).

~/.claude/hooks/statusline.sh 
#!/usr/bin/env bash
# Claude Code statusLine (CLI 2.1.x). Reads session JSON on stdin.
# Schema verified against bundled cli.js 2.1.109:
#   model.display_name, workspace.current_dir,
#   cost.total_cost_usd,
#   context_window.{used_percentage, remaining_percentage,
#                   current_usage, context_window_size},
#   exceeds_200k_tokens.

set -u
JSON="$(cat)"
get() { printf '%s' "$JSON" | jq -r "$1 // empty" 2>/dev/null; }

MODEL=$(get '.model.display_name')
DIR=$(get '.workspace.current_dir')
COST=$(get '.cost.total_cost_usd')
USED_PCT=$(get '.context_window.used_percentage')
OVER=$(get '.exceeds_200k_tokens')

BASE="${DIR##*/}"; [[ -z "$BASE" ]] && BASE="$DIR"

CTX="🟢"
if [[ -n "$USED_PCT" && "$USED_PCT" != "null" ]]; then
  PCT_INT=$(printf '%.0f' "$USED_PCT" 2>/dev/null || echo "")
  if [[ "$PCT_INT" =~ ^[0-9]+$ ]]; then
    if   (( PCT_INT >= 85 )); then CTX="🔴 ${PCT_INT}%"
    elif (( PCT_INT >= 65 )); then CTX="🟡 ${PCT_INT}%"
    else                           CTX="🟢 ${PCT_INT}%"
    fi
  fi
elif [[ "$OVER" == "true" ]]; then
  CTX="🔴 >200k"
fi

COST_STR=""
[[ -n "$COST" ]] && COST_STR=$(printf '$%.3f' "$COST" 2>/dev/null || echo "\$$COST")

printf '%s | %s | %s' "${MODEL:-?}" "$BASE" "$CTX"
[[ -n "$COST_STR" ]] && printf ' | %s' "$COST_STR"

Installation (bash / macOS / WSL)

terminal 
chmod +x ~/.claude/hooks/statusline.sh

Then add to ~/.claude/settings.json:

~/.claude/settings.json 
{
  "statusLine": {
    "type": "command",
    "command": "bash $HOME/.claude/hooks/statusline.sh"
  }
}

Restart your terminal. Every session will now show:

example output 
Sonnet 4.6 | gymos | 🟢 32% | $0.470
Sonnet 4.6 | gymos | 🟡 78% | $1.204
Sonnet 4.6 | gymos | 🔴 91% | $2.381

Windows PowerShell variant

Use pwsh.exe (PowerShell 7), never powershell.exe (5.1) — 5.1 has tokenizer edge cases that break some scripts.

~/.claude/hooks/statusline.ps1 
$json = [Console]::In.ReadToEnd() | ConvertFrom-Json
$model = $json.model.display_name
$dir   = Split-Path -Leaf $json.workspace.current_dir
$pct   = [int]$json.context_window.used_percentage
$cost  = $json.cost.total_cost_usd

$ctx = if ($pct -ge 85) { "🔴 $pct%" }
       elseif ($pct -ge 65) { "🟡 $pct%" }
       else { "🟢 $pct%" }

"$model | $dir | $ctx | `$$([math]::Round($cost,3))"

Register in settings.json:

~/.claude/settings.json 
{
  "statusLine": {
    "type": "command",
    "command": "pwsh -File $HOME/.claude/hooks/statusline.ps1"
  }
}
